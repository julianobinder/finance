# Contributing to the Project

We welcome contributions to this project! Because we license the software under the **GNU Affero General Public License v3.0 (AGPL-3.0)**, we have specific legal requirements for accepting contributions.

## How to Contribute

1. **Fork the Repository**: Create a fork of this repository on GitHub.
2. **Make Improvements**: Make your changes in a new branch in your fork.
3. **Commit Your Changes**: Ensure your code is clean and adheres to the project standards.
4. **Agree to the CLA**: Read the [Contributor License Agreement](CLA.md). When you submit a Pull Request, you must check the box indicating you agree to assign the copyright of your contribution to the original repository owner (Juliano Binder).
5. **Submit a Pull Request**: Submit your Pull Request back to the main branch of this repository.

## The Contribution Loop (Why We Do This)

Under the terms of the AGPL-3.0, any modified versions of this software hosted over a network or distributed to others must have their source code released under the same license. To maintain the health of the project, we ask that all improvements be upstreamed (committed back) to this original repository.

Agreeing to the [CLA](CLA.md) ensures the project owner (Juliano Binder) maintains the ability to license the software under different terms or commercialize it, while keeping the repository open-source.
